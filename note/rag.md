# RAG

检索增强生成

切分->向量化->检索->重排->生成

## 文档拆分

按语义分，不要只按固定长度硬切。保留标题，段落、来源路径等元数据

## 向量索引

把chunk编码进向量库，检索时先召回，再按相关性排序

## Embedding（向量嵌入）

RAG 的搜索不是关键词匹配，而是语义搜索。原理是把文字变成一串数字（向量），语义相近的文字，数字也接近。

搜索时把问题也变成向量，找最接近的文档片段，就是语义搜索。

```json
// 1.建库
原始文档
    ↓
切块（把长文档切成小段，每段 500 字左右）
    ↓
每块文字 → 调用 Embedding API → 变成向量
    ↓
把"文字 + 向量"存入向量数据库
// 2.查询
用户问题 → 变成向量
    ↓
在向量数据库里找最相似的几块文字
    ↓
把这几块文字 + 用户问题组合成 prompt
    ↓
发给 Claude → 得到回答
```

常用的Embedding：

- sentence-transformers
- Voyage
- OpenAI

向量数据库

- Chroma（本地）
- Pinecone（云端）
